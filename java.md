# Java 주석 총정리

## Java란 무엇인가?

Java는 객체지향 프로그래밍 언어 중 하나로, 다양한 운영체제에서 실행할 수 있도록 설계된 언어입니다.

쉽게 말하면:

- Java는 프로그램을 만들기 위한 언어입니다.
- 웹, 앱, 서버, 안드로이드, 기업용 프로그램 등 여러 분야에서 사용됩니다.
- 한 번 작성한 코드를 여러 환경에서 비교적 동일하게 실행할 수 있는 것이 큰 특징입니다.

---

## Java의 특징

### 1. 객체지향 언어

Java는 객체지향 프로그래밍 언어입니다.

객체지향이란:

- 데이터와 기능을 하나로 묶어서 관리하는 방식
- 현실 세계의 사물을 코드로 표현하기 쉬운 방식

예를 들어:

- 학생 → `Student` 클래스
- 계좌 → `BankAccount` 클래스
- 자동차 → `Car` 클래스

이처럼 Java는 클래스를 만들고, 그 클래스로 객체를 만들어 프로그램을 구성합니다.

### 2. 운영체제에 덜 종속적

Java는 Windows, macOS, Linux 같은 다양한 운영체제에서 실행할 수 있습니다.

이유는 Java 코드가 바로 기계어로 실행되는 것이 아니라, 중간 형태인 바이트코드로 변환된 뒤 JVM이 실행하기 때문입니다.

그래서 자주 나오는 표현이:

`Write Once, Run Anywhere`

즉:

- 한 번 작성하고
- 여러 환경에서 실행 가능

이라는 뜻입니다.

### 3. 배우기 비교적 좋은 문법 구조

Java는 문법이 완전히 쉬운 언어는 아니지만, 구조가 비교적 명확하고 체계적입니다.

- 자료형
- 메서드
- 클래스
- 객체
- 상속
- 예외처리

이런 개념들이 분명하게 나뉘어 있어서 프로그래밍 기초를 배우기에 좋습니다.

### 4. 라이브러리가 풍부함

Java는 기본 제공 기능이 많습니다.

예:

- 문자열 처리
- 수학 계산
- 파일 입출력
- 네트워크
- 컬렉션
- 예외 처리
- 스레드

그래서 복잡한 기능도 기본 라이브러리로 많이 해결할 수 있습니다.

---

## Java 프로그램은 어떻게 실행될까?

Java는 보통 아래 순서로 실행됩니다.

1. 사람이 `.java` 파일에 코드를 작성
2. `javac`로 컴파일
3. `.class` 바이트코드 파일 생성
4. `java` 명령으로 JVM이 실행

예:

```java
public class Main {
    public static void main(String[] args) {
        System.out.println("Hello Java");
    }
}
```

컴파일:

```bash
javac Main.java
```

실행:

```bash
java Main
```

여기서:

- `Main.java` = 사람이 작성하는 소스 코드
- `Main.class` = 컴파일된 바이트코드
- JVM = `.class` 파일을 실행하는 환경

---

## JDK, JRE, JVM이란?

Java를 공부하면 꼭 나오는 3가지입니다.

### JVM

`JVM`은 `Java Virtual Machine`의 줄임말입니다.

역할:

- Java 바이트코드를 읽어서 실행함
- 운영체제 위에서 Java 프로그램이 동작하게 해 줌

쉽게 말하면:

- Java 프로그램을 실행해 주는 엔진

### JRE

`JRE`는 `Java Runtime Environment`의 줄임말입니다.

역할:

- Java 프로그램을 실행하는 데 필요한 환경
- JVM과 기본 실행 라이브러리를 포함

쉽게 말하면:

- 실행용 Java 묶음

### JDK

`JDK`는 `Java Development Kit`의 줄임말입니다.

역할:

- Java 프로그램을 개발할 때 필요한 도구 모음
- JRE를 포함
- `javac` 같은 컴파일러도 포함

쉽게 말하면:

- 개발용 Java 묶음

정리:

- JVM = 실행 엔진
- JRE = 실행 환경
- JDK = 개발 도구 + 실행 환경

## Java에서 자주 보는 기본 구조

기본 예제:

```java
public class Main {
    public static void main(String[] args) {
        System.out.println("안녕하세요");
    }
}
```

이 코드의 의미:

- `class` : 코드를 담는 틀
- `Main` : 클래스 이름
- `main` : 프로그램 시작 메서드
- `System.out.println()` : 화면에 출력하는 자바 기본 메서드

즉 Java 프로그램은 보통 클래스 안에 메서드를 넣어서 작성합니다.

## 1. 주석이란?

주석은 사람이 코드를 이해하기 쉽도록 설명을 적어 두는 글입니다.  
주석은 프로그램 실행에는 직접 영향을 주지 않습니다.

주석을 쓰는 이유:

- 코드의 의미를 설명하기 위해
- 왜 이렇게 작성했는지 기록하기 위해
- 다른 사람이나 미래의 내가 코드를 이해하기 쉽게 하기 위해
- 문서화하기 위해

---

## 2. 자바 주석의 종류

자바에는 크게 3가지 주석이 있습니다.

1. 한 줄 주석
2. 여러 줄 주석
3. Javadoc 주석

---

## 3. 한 줄 주석

형식:

```java
// 주석 내용
```

설명:

- `//` 뒤의 내용은 그 줄 끝까지 주석 처리됩니다.
- 가장 자주 쓰는 기본 주석입니다.

예:

```java
// 정수형 변수 선언
int age = 20;
```

여러 줄에 각각 한 줄 주석을 붙일 수도 있습니다.

```java
// 이름을 저장합니다.
// 문자열은 큰따옴표로 감쌉니다.
String name = "홍길동";
```

언제 많이 쓰나:

- 코드 한 줄 설명
- 짧은 메모
- 예제 설명

---

## 4. 여러 줄 주석

형식:

```java
/* 주석 내용 */
```

또는

```java
/*
여러 줄에 걸친
주석 내용
*/
```

설명:

- `/*` 로 시작하고 `*/` 로 끝납니다.
- 여러 줄 설명을 적을 때 사용합니다.

예:

```java
/*
이 코드는 학생 점수를 계산하는 예제입니다.
합계와 평균을 함께 구합니다.
*/
int total = score1 + score2 + score3;
```

주의:

- 여러 줄 주석 안에 또 `/* */`를 중첩해서 넣을 수는 없습니다.

---

## 5. Javadoc 주석

형식:

```java
/**
 * 설명 내용
 */
```

설명:

- `/**` 로 시작하는 특별한 주석입니다.
- 클래스, 메서드, 생성자, 필드 등을 문서처럼 설명할 때 사용합니다.
- 일반 주석보다 공식 문서화에 더 가깝습니다.

예:

```java
/**
 * 두 수를 더한 결과를 반환합니다.
 *
 * @param a 첫 번째 수
 * @param b 두 번째 수
 * @return 두 수의 합
 */
public static int add(int a, int b) {
    return a + b;
}
```

이 주석은 보통 메서드 선언 바로 위에 씁니다.

---

## 6. Javadoc에서 자주 쓰는 태그

### `@param`

매개변수 설명

```java
/**
 * @param name 사용자 이름
 */
```

### `@return`

반환값 설명

```java
/**
 * @return 계산 결과
 */
```

### `@throws`

발생 가능한 예외 설명

```java
/**
 * @throws IOException 파일 읽기 실패 시 발생
 */
```

### `@author`

작성자 설명

```java
/**
 * @author USER
 */
```

### `@since`

언제부터 추가되었는지 설명

```java
/**
 * @since 1.0
 */
```

---

## 7. Javadoc 안에서 자주 쓰는 문법

### `{@code ...}`

코드처럼 표시할 때 사용합니다.

```java
/**
 * {@code add(1, 2)}는 3을 반환합니다.
 */
```

### `{@link ...}`

다른 클래스나 메서드를 연결해서 설명할 때 사용합니다.

```java
/**
 * 자세한 내용은 {@link Math} 클래스를 참고하세요.
 */
```

### `<p>`

문단을 구분할 때 사용합니다.

```java
/**
 * 첫 번째 설명입니다.
 * <p>
 * 두 번째 문단 설명입니다.
 */
```

즉 Javadoc은 Markdown이 아니라 Javadoc 전용 문법과 일부 HTML을 사용합니다.

## 8. 주석 예제 비교

### 한 줄 주석 예제

```java
// 숫자를 저장하는 변수입니다.
int number = 10;
```

### 여러 줄 주석 예제

```java
/*
학생 정보를 저장하는 코드입니다.
이름과 나이를 저장합니다.
*/
String name = "철수";
int age = 20;
```

### Javadoc 예제

```java
/**
 * 학생 이름을 출력합니다.
 *
 * @param name 출력할 이름
 */
public static void printName(String name) {
    System.out.println(name);
}
```

## 9. 주석과 어노테이션의 차이

주석과 어노테이션은 다릅니다.

### 주석

- 설명을 적는 것
- 실행 자체에는 직접 영향이 없음

예:

```java
// 학생 이름 출력
```

### 어노테이션

- 자바에게 특별한 의미를 알려 주는 표시
- 컴파일러나 프레임워크가 읽을 수 있음

예:

```java
@Override
public String toString() {
    return "학생";
}
```

정리:

- `//`, `/* */`, `/** */` = 주석
- `@Override`, `@Deprecated` = 어노테이션

## 10. 좋은 주석과 나쁜 주석

### 좋은 주석

- 코드만 봐서는 알기 어려운 이유를 설명함
- 왜 이렇게 작성했는지 알려 줌
- 중요한 규칙이나 주의점을 적어 둠

예:

```java
// 0으로 나누는 상황을 막기 위해 먼저 검사합니다.
if (number != 0) {
    result = 10 / number;
}
```

### 나쁜 주석

- 코드만 봐도 뻔한 내용을 반복함
- 실제 코드와 다르게 오래된 설명이 남아 있음

예:

```java
// number에 10을 저장한다.
int number = 10;
```

이런 주석은 너무 단순해서 큰 도움이 안 될 수 있습니다.

## 11. 주석을 쓰는 위치

### 코드 위

가장 일반적인 방식입니다.

```java
// 학생 수를 저장합니다.
int studentCount = 30;
```

### 코드 옆

짧은 보충 설명에 사용합니다.

```java
int max = 100; // 최대 점수
```

### 메서드 위

Javadoc이나 중요한 설명을 적을 때 사용합니다.

```java
/**
 * 두 수를 더합니다.
 */
public static int add(int a, int b) {
    return a + b;
}
```

## 12. 주석으로 코드 막기

임시로 실행을 막고 싶을 때 주석 처리할 수 있습니다.

예:

```java
// System.out.println("이 코드는 실행되지 않습니다.");
```

또는

```java
/*
System.out.println("첫 번째 줄");
System.out.println("두 번째 줄");
*/
```

하지만 너무 오래 주석 처리된 코드를 남겨 두는 것은 좋지 않습니다.

---

## 13. 실무에서 주석을 쓰는 기준

보통 아래처럼 생각하면 됩니다.

- 코드 자체가 명확하면 주석이 없어도 됨
- 이유가 복잡하면 주석을 씀
- 공개 API나 중요한 메서드에는 Javadoc을 씀
- 임시 메모는 너무 오래 남기지 않음

---

## 14. 자바 학습 기준으로 기억할 핵심

- `//` : 한 줄 주석
- `/* ... */` : 여러 줄 주석
- `/** ... */` : Javadoc 주석
- Javadoc은 메서드나 클래스 설명용
- `@param`, `@return`, `@throws`를 자주 사용함
- `@Override`는 주석이 아니라 어노테이션임

---

## 15. 짧은 정리

자바의 주석은 코드를 설명하기 위한 도구입니다.  
가장 많이 쓰는 것은 `//` 주석이고, 문서처럼 정리할 때는 `/** ... */` 형태의 Javadoc을 사용합니다.  
실무나 공부에서는 "코드가 무엇을 하는지"보다 "왜 이렇게 작성했는지"를 설명하는 주석이 더 가치 있습니다.
